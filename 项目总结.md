# 英语表达练习 — 项目总结

## 起点：发现了什么问题

学英语的人每天都在做三件事：**查翻译、看例句、写日记**。但传统工具把这三件事割裂了：

| 场景 | 通常做法 | 问题 |
|------|---------|------|
| 想说一句话但不会用英语表达 | 打开有道翻译 | 翻译完划走，过两天又忘了 |
| 早上刷到「每日说三句」觉得很好 | 截图收藏 | 图片散落相册，想复习找不到 |
| 想练英文日记 | 写在备忘录里 | 写了没人改，不知道语法对不对 |

核心矛盾：**输入、内化、输出是断开的。** 翻译是翻译，收藏是收藏，写作是写作，三者没有形成闭环。

## 产品思路：输入 → 内化 → 输出

我设计了一个「三合一」的英语练习工具，把三个环节串起来：

```
输入（语料库）→ 内化（翻译笔记）→ 输出（日记批改）
    ↑                                      ↓
    └———— 所有内容可导出 PDF/Word ←—————————┘
```

**输入层**：看到好句子随时收藏，AI 自动分类，一键翻译中文释义，形成个人语料库。  
**内化层**：遇到卡壳的中文不再是翻译完就忘，而是自动保存到历史记录，随时搜索回顾。  
**输出层**：每天写英文日记，AI 像老师一样用「红笔」批改语法，告诉你哪里不对、为什么、怎么改。

## 迭代过程

### V1.0 — 从零到一（一个 HTML 文件）

第一个版本就是一个 `index.html`，纯 HTML/CSS/JavaScript，双击打开就能用。选择这个方式的原因是：
- 零依赖，不需要安装任何东西
- 数据存浏览器本地存储（localStorage），隐私安全
- 功能不受限，翻译、日记、语料库三模块齐全

缺点：数据只在一台设备上，手机和电脑无法互通。代码结构简单但维护困难。

### V1.1 — 工程化重构（React + Vite）

把整个项目用 React 18 重写，搭配 Vite 6 构建工具：

- 组件化拆分：Header、TranslateTab、DiaryTab、CorpusTab 各自独立
- 工具函数模块化：API 调用、数据存储、导出逻辑分离
- 支持 PWA（渐进式网页应用），手机浏览器打开可添加到桌面，像原生 App 一样全屏运行
- 部署到 GitHub Pages，免费、全球可访问、自动 HTTPS

技术栈：React、Vite、CSS 变量、localStorage、Service Worker

### V1.2 — 自建后端（多设备同步）

纯前端的问题在于数据存浏览器里，换设备就看不到。于是我搭建了一个后端服务：

- **后端框架**：Node.js + Express（轻量级 HTTP 服务）
- **数据库**：SQLite（单个文件，不需要安装数据库软件，备份就是复制文件）
- **部署**：阿里云 ECS 云服务器，pm2 进程守护（崩溃自动重启、开机自启）
- **安全**：简易密码鉴权，API Key 只存服务器端，前端不再暴露密钥
- **容错**：服务器不可用时，前端自动降级到本地存储模式，不影响使用

架构图：
```
手机 / 电脑 → GitHub Pages（前端，免费托管）
                    ↓ API 调用（密码鉴权）
            阿里云 ECS :8080（后端，月付几十块）
                    ↓
            DeepSeek API（AI 能力） + SQLite（数据存储）
```

### V1.3 — 细节打磨

- **双向翻译**：中→英和英→中一键切换
- **红笔批改**：日记批改不是只给一个修改版，而是逐条列出「原文 → 修改后 → 原因」，像老师改作业
- **AI 自动分类**：收藏句子时 AI 根据内容自动归类（职场/日常/社交等）
- **导出自由**：所有数据可导出 PDF 和 Word，不锁定用户
- **连续打卡**：日记自动计算连续写作天数，形成正向激励

## 技术关键词解释

| 术语 | 通俗解释 |
|------|---------|
| React | Facebook 开源的前端框架，把页面拆成可复用的「组件」 |
| Vite | 新一代前端构建工具，启动快、打包快 |
| localStorage | 浏览器自带的微型数据库，数据存在用户自己电脑里 |
| PWA | 网页可以像 App 一样添加到手机桌面，离线也能打开 |
| GitHub Pages | GitHub 提供的免费网页托管服务 |
| ECS | 阿里云的云服务器，相当于一台永远开机的远程电脑 |
| Node.js | 让 JavaScript 可以在服务器上运行的环境 |
| Express | Node.js 最流行的 Web 框架，几行代码就能搭建 API |
| SQLite | 轻量级数据库，整个数据库就是一个文件 |
| API | 前后端通信的接口，前端发送请求，后端返回数据 |
| API Key | 调用 AI 服务的密钥，类似「门禁卡」 |
| pm2 | 进程管理工具，服务挂了自动重启 |
| DeepSeek | 国内的 AI 大模型，提供翻译、文本润色等能力 |
