# 英语表达练习 — 产品需求文档 (PRD)

## 一句话定位

一个**输入→内化→输出**三位一体的英语表达提升工具，让每一次翻译都变成积累，让每一篇日记都能得到反馈。

## 为什么要做这个

市面上学英语的工具各有侧重，但缺乏一个打通闭环的产品：

| 场景 | 现有工具 | 痛点 | 本产品 |
|------|---------|------|--------|
| 遇到不会说的中文 | 有道/Google 翻译 | 翻译完就忘，没有笔记 | 翻译 + 自动保存到历史记录，随时回顾 |
| 练习写作 | Word/Grammarly | 无反馈或反馈生硬 | AI 批改语法 + 润色为地道表达 |
| 年初立 flag 写日记 | Notion/备忘录 | 写了没反馈，坚持不下去 | 每天写 + AI 实时批改 + 连续天数激励 |
| 刷到好句子想记 | 截图/微信收藏 | 散落各处，找不到 | 统一语料库 + AI 自动分类 |
| 想系统复习 | Anki | 制卡成本高 | 语料库即题库（未来） |

**核心理念：不靠死记硬背，靠「输入 → 内化 → 输出」的闭环。**

- **输入**（语料库）：早上看到好句子 → 收藏 → AI 分类 → 一键翻译
- **内化**（翻译练习）：遇到卡壳中文 → 翻译 → 保存 → 回顾
- **输出**（英文日记）：每天写 → AI 批改 → 地道表达 → 持续进步

## 用户画像

- 工作中需要用到英语，但口语/书面表达不够地道
- 每天刷微信公众号「每日说三句」等英语学习内容
- 想写英文日记但坚持不下来，写了也不知道对不对
- 收藏了很多英语表达但散落在各处，从不复习

## 产品原则

1. **本地优先** — 数据归用户所有，不上传服务器
2. **AI 辅助而非替代** — AI 给建议，用户做决定（翻译可修改、批改可拒绝）
3. **低摩擦输入** — 收藏一个表达 ≤ 10 秒
4. **即时正反馈** — streak、数量增长、批改对比，用户能感受到进步
5. **导出自由** — 所有数据随时导出 PDF/Word，不锁定用户

## 数据流

```
用户输入 → DeepSeek API → 显示结果 → 用户确认 → localStorage 持久化
                                                    ↓
                                           可用于复习、导出
```

## 技术约束

- 纯前端，零后端（API key 存 localStorage）
- 不收集用户数据
- 单个 HTML 文件也能跑（HTML 版），React 版需要 `npm install`
- GitHub Pages / Vercel 可部署

## 对标分析

| 产品 | 差异 |
|------|------|
| 有道翻译 | 无笔记、无历史管理、无导出 |
| Grammarly | 无日记、无翻译记录、付费 |
| 墨墨背单词 | 只能背单词，不涉及表达句子 |
| Notion | 无 AI 批改、无翻译、分类需手动 |
| Anki | 制卡成本高、无 AI、无写作 |

## 愿景

3 个月后，打开你的语料库，里面有 200 条地道表达；
翻看你的日记，能看到 AI 批改的红色越来越少；
遇到不会说的中文，不是翻译完就忘，而是变成语料库里的一条记录。

英语练习，不只是翻译。
